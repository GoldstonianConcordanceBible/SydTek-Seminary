# Purpose

This document defines how SydTek Seminary credentials are verified.

---

# Verification Methods

Credentials may be verified through:

- Minister Registry entry
- Credential Verification ID
- Digital certificate
- Portfolio link
- QR code
- GitHub record
- Optional on-chain credential reference

---

# Verification ID Format

```text
STM-CV-[YEAR]-[SEQUENCE]

Verification Record Fields

Each verification record may include:

* Verification ID
* Recipient name
* Credential title
* Credential level
* Issue date
* Status
* Portfolio evidence link
* Registry ID
* Issuer
* Review authority

⸻

Status Types

* Active
* Inactive
* Retired
* Suspended
* Revoked
* Expired

⸻

Privacy Standard

No sensitive personal data should be stored in public verification records.

Public records should only include information necessary for credential verification.

⸻

Principle

Credential verification exists to protect trust, transparency, and accountability.
