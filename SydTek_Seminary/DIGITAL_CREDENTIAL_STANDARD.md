# Purpose

This standard defines how SydTek Seminary issues, verifies, and manages digital ministry credentials.

Digital credentials support transparency, portability, portfolio-based review, and public verification.

---

# Credential Types

SydTek Seminary may issue:

- Digital Certificates
- Digital Badges
- Ministry Completion Credentials
- Licensed Minister Credentials
- Ordained Minister Credentials
- Senior Minister Credentials
- Optional On-Chain Credentials

---

# Required Credential Fields

Each credential should include:

- Recipient Name
- Credential Title
- Issuing Body
- Issue Date
- Credential ID
- Credential Level
- Verification Status
- Portfolio Evidence Link

---

# Credential ID Format

Example:

```text
STM-DC-2026-0001

Verification Methods

Credentials may be verified through:

* Seminary Minister Registry
* Digital Certificate Link
* QR Code
* GitHub Verification Record
* Credential Verification Page
* Optional On-Chain Reference
* W3ID Integration

⸻

Portfolio Evidence

All major credentials should connect to documented evidence.

Acceptable evidence includes:

* Courses completed
* Sermons
* Bible studies
* Books
* Articles
* Research
* Public teaching
* Ministry service
* Community service
* Digital ministry
* Leadership experience

⸻

Credential Status Types

* Active
* Inactive
* Retired
* Suspended
* Revoked
* Expired

⸻

Optional On-Chain Credentialing

SydTek Seminary may use blockchain verification for optional credential transparency.

On-chain credentials may include:

* Credential ID
* Recipient DID or W3ID
* Issue Date
* Issuer Verification
* Hash of Credential Record
* Portfolio Reference

No private personal data should be stored directly on-chain.

⸻

Governance

Digital credentials are governed by:

* Seminary Board
* Digital Credential Committee
* Ordination Review Committee

All credentials must be reviewed and issued according to documented Seminary standards.
